# 题库模板设计说明

## 1. 概述

本文档详细说明了试题管理系统中题库导入模板的设计规范。该模板采用纯文本格式，支持多种题型、双语题目以及丰富的元数据信息，便于与其他数据库系统对接。

## 2. 基本格式规范

### 2.1 题目标识符
- 每道题以 `[题型]` 开头，题型标识符用方括号包围
- 题型后可跟答案标识符（如 `[A]`、`[ABD]` 等）
- 题目内容紧随其后

### 2.2 答案格式
- 选择题答案紧跟在题型标识后，用方括号包围
- 单选题：`[A]`、`[B]`、`[C]`、`[D]` 等
- 多选题：`[ABCD]` 等，多个选项连续排列
- 是非题：`[是]`、`[否]` 或 `[A]`、`[B]`

### 2.3 选项格式
- 选择题选项格式：`[选项字母]` + 选项内容
- 选项字母用大写英文字母表示

## 3. 题型设计

### 3.1 单选题
```
[单选][D]
湖北省的省会城市是（）？
[A]长沙
[B]宜昌
[C]荆州
[D]武汉
<解析>
这是一段解析
武汉是湖北省省会城市
<解析>
```

**设计要点：**
- 题型标识：`[单选]`
- 答案格式：`[选项字母]`
- 解析可选，用 `<解析>...</解析>` 包围

### 3.2 多选题
```
[多选][ABD]
请问下列哪些书籍是中国四大小说名著？
[A]三国演义
[B]红楼梦
[C]老残游记
[D]水浒传
<解析>
老残游记不属于四大名著，其他都属于
<解析>
```

**设计要点：**
- 题型标识：`[多选]`
- 答案格式：`[选项字母组合]`，如 `[ABD]`
- 支持多个正确选项

### 3.3 是非题
```
[是非][B]
在《西游记》中六耳猕猴是唐僧的徒弟吗？
[A]是
[B]否
<解析>
老残游记不属于四大名著，其他都属于
<解析>
```

**设计要点：**
- 题型标识：`[是非]`
- 答案格式：`[A]`/`[B]` 或 `[是]`/`[否]`

### 3.4 简答题（通用题型）
```
[简答]
你对今天这节课有什么评价？
<参考答案>
这是一段参考答案
课程非常的生动
<参考答案>
```

**设计要点：**
- 题型标识：`[简答]`
- 参考答案用 `<参考答案>...</参考答案>` 包围

### 3.5 扩展题型（简答变体）
```
[简答>计算]
（英语）
A tract of 10 hectares has been harvested...
<参考答案>
1.6
<参考答案>
<解析>
（400*25*10*0.4-300*20*10*0.4)/(10*1000)=1.6
<解析>
```

**设计要点：**
- 题型标识：`[简答>具体题型]`，如 `[简答>计算]`、`[简答>论述]`
- 支持论述、计算、材料分析等具体题型
- 通过这种方式实现与其他数据库的对接

## 4. 双语支持机制

### 4.1 双语题目格式
```
[简答>论述]
（中文）
什么是代际公平？...
（英文）
What is intergenerational equity?...
<参考答案>
定义代际公平(2分); 分析在森林管理中的体现(4分);...
<参考答案>
```

**设计要点：**
- 中文内容用 `（中文）` 标注
- 英文内容用 `（英文）` 标注
- 中英文内容在同一题目中并列显示
- 答案和解析统一使用中文

### 4.2 语言标注规则
- 语言标注必须用中文括号 `（）` 包围
- 语言标注应放在题目内容的上方
- 支持仅中文、仅英文或中英双语三种模式

## 5. 元数据结构

### 5.1 解析信息
```
<解析>
知识点: Intergenerational equity
标签: equity,sustainability,forest management
<解析>
```

**设计要点：**
- 解析内容用 `<解析>...</解析>` 包围
- 可包含知识点、标签等元数据信息
- 解析为可选内容，不是每道题都必须有

### 5.2 参考答案
```
<参考答案>
定义代际公平(2分); 分析在森林管理中的体现(4分);...
<参考答案>
```

**设计要点：**
- 参考答案用 `<参考答案>...</参考答案>` 包围
- 适用于简答题、计算题、论述题等需要参考答案的题型

## 6. 模板设计优势

### 6.1 兼容性
- 纯文本格式，兼容各种数据库系统
- 简单的标记语法，易于解析和处理

### 6.2 扩展性
- 通过 `[简答>具体题型]` 格式支持任意题型扩展
- 支持丰富的元数据信息

### 6.3 国际化
- 内置双语支持机制
- 灵活的语言标注方式

### 6.4 易用性
- 格式直观，便于人工编辑
- 结构清晰，便于批量处理

## 7. 实现建议

### 7.1 解析器设计
- 逐行解析，识别题型标识符
- 使用正则表达式匹配各种标记格式
- 支持多行内容解析（如题目内容、解析等）

### 7.2 数据验证
- 验证题型标识符的有效性
- 验证答案格式的正确性
- 验证双语标注的规范性

### 7.3 错误处理
- 提供详细的错误信息和行号
- 支持部分导入，跳过错误题目
- 提供格式验证工具

## 8. 使用示例

### 8.1 完整题目示例
```
[简答>材料分析]
（中文）
阅读以下材料，回答问题：
材料内容...
（英文）
Read the following material and answer the questions:
Material content...
<参考答案>
参考答案内容...
<参考答案>
<解析>
知识点: ...
标签: ...
<解析>
```

### 8.2 系统对接示例
- 其他数据库导出时，将论述题、计算题、材料题等转换为 `[简答>具体题型]` 格式
- 保持原有的知识点、标签等元数据信息
- 确保双语内容的正确标注

## 9. 注意事项

1. **格式一致性**：确保所有题目遵循相同的格式规范
2. **编码问题**：使用 UTF-8 编码以支持中文字符
3. **换行处理**：正确处理多行题目内容和答案
4. **特殊字符**：避免在题目中使用与标记冲突的特殊字符
5. **验证机制**：导入前进行格式验证，确保数据完整性
